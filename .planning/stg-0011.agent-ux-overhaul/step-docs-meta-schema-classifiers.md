# stg-0011 — docs-meta-schema-classifiers

## Цель

Если требуется — расширить `docs/.docswalker/meta-schema.yml` под новую семантику multi-tree-классификаторов: поддержка required tree-связи с возможностью указать `none` как явную категорию. Если текущая meta-schema уже это поддерживает (tree-связь всегда required=true, целевой узел — любой того же tree), правок не требуется и шаг закрывается без изменений в файле.

## Файлы

`docs/.docswalker/meta-schema.yml` — узкое исключение из правила «доступ к docs только через DocsWalker» (служебный файл, см. CLAUDE.md).

## Действия

1. Прочитать текущую meta-schema через `Read` (исключение разрешено).
2. Проверить, покрывает ли текущая модель `tree-связь типа классификатор с целевым узлом `none``: tree-refs всегда required+cardinality=one, что подходит. Проверить, нужен ли отдельный признак, отличающий «классификатор-дерево» от «структурного» (типа path).
3. Если правки не нужны — зафиксировать решение в чате, шаг закрыть без изменений.
4. Если правки нужны — внести минимальные (по образцу существующих расширений), сохранить версию meta-schema.

## Риски

Правка meta-schema затрагивает базу всей валидации. До правки — снапшот текущего файла. Если ломается `check-integrity` после правки — откат.
