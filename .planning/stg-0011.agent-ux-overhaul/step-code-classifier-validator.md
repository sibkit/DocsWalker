# stg-0011 — code-classifier-validator

## Цель

Расширить валидатор kernel: для каждой required tree-связи на атом-типе проверять, что у узла есть ровно одна ссылка в этом дереве. Целевой узел — любая категория или `none`. Отсутствие классификатора → ошибка `missing_classifier` с `hint`, перечисляющим доступные категории.

## Файлы

Через `mcp__glider__find_code`:
- Kernel: Validator (точное имя — после Glider-поиска).
- Возможно — отдельный `ClassifierValidator.cs`.

## Действия

1. Изучить текущую логику валидатора (как проверяется required-связи на атом-типах).
2. Расширить: для каждого declared tree-scope с required tree-ref на типе — обязательная проверка наличия.
3. Сообщение ошибки `missing_classifier` с полями `node_id`, `tree_name`, `available_categories` (через `hint`).
4. Unit-тесты на валидатор: узел без классификатора, узел с `none`, узел с реальной категорией, узел с двумя ссылками в одном tree (ошибка).

## Риски

После расширения валидатора `check-integrity` на текущем 337-узловом графе вернёт массовые ошибки до миграции данных. Это ожидаемо.
