# stg-0001 — yaml-emitter

## Цель
Реализовать собственный YAML-сериализатор под стилевые правила `docs/`: 2-space indent, минимум кавычек, склейка id+title по `title_format`. Эмиттер используется всеми write-операциями.

## Файлы
`src/DocsWalker.Core/Yaml/Emitter.cs` — emitter; принимает узел или весь документ, выдаёт текст YAML
`src/DocsWalker.Core/Yaml/Quoting.cs` — политика кавычек: когда оставлять без кавычек, когда одинарные, когда двойные

## Действия
1. Описать модель входа: `Graph`, документ или поддерево узлов.
2. Реализовать обход и вывод:
   - block-mapping и block-sequence — основной стиль;
   - flow-sequence для коротких списков по правилам оформления;
   - single_key_mapping (склейка ключа по `title_format`).
3. Политика кавычек:
   - по умолчанию без кавычек;
   - одинарные — если значение содержит спецсимволы YAML или начинается с `#`/`-`/числа в неподходящем контексте;
   - двойные — только при необходимости escape-последовательностей.
4. Не выводить запрещённые конструкции (`|`, `>`, `&`, `*`, `!`, `%`).
5. Тесты на круглую сериализацию: распарсить `docs/*.yml` → сериализовать → распарсить, сравнить графы.

## Риски
- Стилевые правила в `docs/Правила оформления.yml` могут содержать нюансы, не покрытые черновиком эмиттера. Сверяться с docs синхронно при разработке.
- Edge-case для unquoted-скаляра, начинающегося с `#` или содержащего `: ` — нужна аккуратная политика, иначе результат перестаёт парситься.
