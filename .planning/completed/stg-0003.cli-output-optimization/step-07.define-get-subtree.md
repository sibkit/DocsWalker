# stg-0003 — define-get-subtree

## Цель
Добавить узел-определение `get_subtree` в секцию `(#17) Операции чтения`. Сейчас CLI-команда `get-subtree` живая (есть в `Commands.cs`, упоминается в правилах и примерах docs — `(#280)`, `(#285)`, `(#353)`, `(#356)`), но узла-определения в перечне «Операции чтения» нет — пробел в spec'е.

## Файлы
`docs/DocsWalker.yml` — секция `(#17) Операции чтения`, блок `definitions`.

## Действия

1. Добавить новый definition-узел в `definitions` секции `(#17)` после `(#23) get_by_path` и перед `(#24) get_refs` (чтобы тематически шёл рядом с другими «многоузловыми» чтениями: `get_map`, `get_nodes`, `get_by_path`).
   Id берётся из `docs/.docswalker/sequence.txt` + 1; title — `get_subtree`; type — `definition` (как у соседей). Текст определения должен покрывать:
   - Что возвращает: поддерево узла в указанном tree-scope.
   - Параметры: `id` (обязательный), `tree` (опц., по умолчанию `path`), `depth` (опц., максимальная глубина: 0 — только корень, 1 — корень + один уровень; без параметра — без ограничения), `fields` (опц., whitelist полей через запятую: `id,type,title,text,out_refs,tokens,subtree_tokens`; без параметра — все поля; `id` присутствует всегда).
   - Шейп ответа: каждый узел поддерева несёт `id`, `type`, `title`, плюс при наличии — `text`, `out_refs` (Map<имя, list<id>>), `tokens` (только сам узел), `subtree_tokens` (узел + потомки в результате), `children` (массив дочерних узлов в текущем дереве).
   - Дефолты-опускание (правило `(#301)` уже зафиксировано в секции CLI-интерфейс): `children: []` не сериализуется (отсутствие поля ⇒ узел-лист в результате); `subtree_tokens`, равный `tokens`, не сериализуется (отсутствие поля ⇒ `subtree_tokens = tokens`). Эти правила в text определения не дублировать (живут в `(#301)`), но формулировку про `children`/`subtree_tokens` строить так, чтобы она была согласована с `(#301)` — без явного «всегда присутствует».
2. Обновить `docs/.docswalker/sequence.txt` под выданный id (next free).
3. Проверить, что после правки `check-integrity` проходит (`docs/Схема.yml` уже знает `definition` — новых типов не вводим).

## Риски
- Возможный замеченный смежный пробел: `get_ancestors` (живой CLI, `Commands.cs:99`) тоже не имеет узла-определения в `(#17)`. В scope этого шага его не включаем — отдельный пункт стратегии (или отказ), вынесен на согласование пользователю.
- Правка `sequence.txt` руками запрещена правилом `(#294)`. Здесь это не правка вручную, а согласованная с созданием атома инкрементная выдача id (тот же эффект, что у `create-node`); оформить как часть транзакции правки docs.
