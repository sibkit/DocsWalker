# stg-0003 — define-get-ancestors

## Цель
Добавить узел-определение `get_ancestors` в секцию `(#17) Операции чтения`. Сейчас CLI-команда `get-ancestors` живая (`Commands.cs:99`, реализована в `ReadApi.GetAncestors`), но узла-определения в перечне «Операции чтения» нет — пробел в spec'е, симметричный с `get_subtree`. Замены команде нет: цепочку родителей в произвольном tree-scope одним вызовом не получить (`get-nodes` даёт только одного path-родителя через `out_refs[path]`; для глубины N — N round-trip'ов).

## Файлы
`docs/DocsWalker.yml` — секция `(#17) Операции чтения`, блок `definitions`.

## Действия

1. Добавить новый definition-узел в `definitions` секции `(#17)` сразу после нового узла `get_subtree` (тематически — оба «обход в указанном tree-scope», смежные definition'ы).
   Id берётся из `docs/.docswalker/sequence.txt` + 1, **после** id, выданного `get_subtree` (то есть `get_ancestors` получает следующий за ним id). title — `get_ancestors`; type — `definition` (как у соседей).
   Текст определения должен покрывать:
   - Что возвращает: цепочку родителей узла в указанном tree-scope, от ближайшего родителя к корню дерева. Если узел сам — корень scope (нет scope-ref), результат пуст.
   - Параметры: `id` (обязательный), `tree` (опц., по умолчанию `path`).
   - Шейп ответа: `{tree, ancestors: [...node...]}` — `tree` повторяет имя scope для подтверждения, `ancestors` — массив узлов в порядке от ближайшего родителя к корню (тот же шейп узла, что у `get_nodes`).
   - Замечание про path: для `tree=path` цепочка заканчивается над root (id=0); сам root в результат не включается, поскольку у него нет path-родителя (концептуально root присутствует, но в `_byId` его нет).
2. Обновить `docs/.docswalker/sequence.txt` под итоговый sequence — на 2 единицы вперёд (один id ушёл `get_subtree`, второй — `get_ancestors`).
3. После правки прогнать `check-integrity` — должно быть `ok=true`.

## Риски
- Совмещается с `define-get-subtree`: оба пункта правят одну секцию `(#17)` и оба берут id из `sequence.txt`. В одной пачке выполняется последовательно: сначала `get_subtree` (id = N), затем `get_ancestors` (id = N+1); `sequence.txt` обновляется одной финальной правкой на N+1 → N+2.
