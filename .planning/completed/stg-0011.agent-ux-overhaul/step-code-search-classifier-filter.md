# stg-0011 — code-search-classifier-filter

## Цель

Расширить `search` параметром `in_tree` (тем же, что в `find`): пересечение результатов текстового поиска (BM25) с multi-tree-фильтром. То есть текстовый запрос ограничивается заданным множеством категорий в указанных классификаторах.

## Файлы

- Kernel: `SearchHandler.cs` (после расширения в шаге `code-search-v2-bm25`).
- `DocsWalker.Mcp/Tools/...` — обновить inputSchema для `search`.

## Действия

1. Добавить параметр `in_tree` в `SearchHandler` (массив `{name, under}`).
2. Применять фильтр до или после BM25-ранжирования (зависит от индекса) — по эффективности после, по корректности результата эквивалентно.
3. Обновить inputSchema MCP-tool `search`.
4. Unit-тесты: search с фильтром по subject, по нескольким классификаторам, с `none` как `under`.

## Риски

Несоответствие inputSchema между `find` и `search` для одного и того же параметра `in_tree` — формат должен совпадать.
