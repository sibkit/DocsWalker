# stg-0002 — axes-folders

## Цель
Поддержать тип `folder` на уровне store: подкаталоги внутри `docs/` становятся узлами графа типа `folder` с осью `path`. `root` — синглтон корня `docs/`. Путь к узлу — цепочка по оси `path` через `folder`-узлы и `document`, разделитель `/`.

## Файлы
`src/DocsWalker.Core/Documents/DocumentScanner.cs` (или эквивалент сканера `docs/`):
- Сканировать `docs/` рекурсивно, исключая `.docswalker/`.
- Для каждой подпапки создавать `folder`-узел; id берётся из служебного файла `docs/.docswalker/folders.yml` (см. ниже). Если папка ещё не зарегистрирована — резервируется новый id из sequence и дописывается строка в файл.
- Для каждого файла — `document`-узел.
- `root`-узел (id=0) — синглтон, в YAML не хранится.

`docs/.docswalker/folders.yml` — служебное хранилище id папок:
- Формат — список `{path: "Папка/Подпапка", id: 42}`. Путь — относительный к `docs/`, разделитель `/`.
- При создании папки — добавляется строка.
- При rename папки — правится поле `path`, id сохраняется.
- При удалении папки — строка убирается.
- Файл вместе с YAML-файлами docs/ обновляется атомарно через `AtomicWriter`.
`src/DocsWalker.Core/Documents/SourceFile.cs` — путь файла относительно `docs/` поддерживает разделитель.
`src/DocsWalker.Core/Store/AtomicWriter.cs`:
- Создание `folder`-узла = создание подкаталога.
- Удаление `folder`-узла = удаление подкаталога (только если пуст в графе и на ФС).
- Переименование `folder` (через `update_node` patch.title) = переименование подкаталога; пути всех вложенных документов на ФС синхронизируются.
`src/DocsWalker.Core/Api/PathResolver.cs` — резолв пути формы `"Папка/Подпапка/Документ/Раздел"` в id; разделитель `/`; пустая строка `""` = root.
`tests/DocsWalker.Tests/DocumentScannerTests.cs`, `WriteApiTests.cs`, `PathResolverTests.cs` — тесты.

## Действия
1. Реализовать сканер с поддержкой подкаталогов и чтение/запись `docs/.docswalker/folders.yml`.
2. Расширить `AtomicWriter` под создание/удаление/переименование подкаталогов и обновление `folders.yml` в одной транзакции.
3. Реализовать `PathResolver` под подкаталоги и пустой path для root.
4. Тесты: создать folder в root; folder в folder; document в folder; rename folder синхронизирует пути; удаление folder с непустым содержимым → ошибка.

## Риски
Решение по хранилищу id принято: служебный файл `docs/.docswalker/folders.yml` (см. strategy.md, раздел «Принятые решения по реализации»). При старте шага зафиксировать формат файла в `docs/DocsWalker.yml` (раздел «Многопроцессная запись» или новый — «Служебные файлы»), синхронно с реализацией.
