# stg-0002 — axes-migrate-docs

## Цель
Привести существующие `docs/*.yml` (`Схема.yml`, `Стек.yml`, `Правила оформления.yml`, `DocsWalker.yml`) в полное соответствие новой модели, чтобы `check-integrity` дал зелёный.

## Файлы
Все `docs/*.yml` — финальная сверка:
- Корневой `id` документа сохраняется (документ — узел типа `document`).
- Никаких отдельных корневых блоков `axes` в обычных документах нет — только в `docs/Схема.yml`.
- `out_refs` блоки внутри узлов остаются, но имена ключей теперь должны быть именами осей kind=explicit (не «ref_type»).
- Структурные нарушения, выловленные `check-integrity`, разбираются по одному.

## Действия
1. Запустить `docswalker check-integrity` после завершения предыдущих axes-шагов.
2. Просмотреть список ошибок; для каждой — либо правка YAML (через `docswalker update-node` или `transaction`), либо правка Схемы, если выявлен пробел в ней.
3. Повторять до зелёного результата.
4. Зафиксировать финальное состояние коммитом «migrate docs/ to axes model».

## Риски
В процессе миграции могут всплыть скрытые предположения о форме YAML, не закрытые шагами axes-meta-schema/axes-schema/axes-docswalker-yml. Каждое такое — отдельная мини-правка предыдущих docs-шагов с возвратом к ним; не пытаться обходить через костыли в коде.
