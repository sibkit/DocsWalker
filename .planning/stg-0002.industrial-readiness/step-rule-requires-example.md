# stg-0002 — rule-requires-example

## Цель
Привести живые `docs/*.yml` в соответствие со Схемой R13: каждое `rule` обязано иметь связь `examples` с минимум одним `example` в качестве цели. Без этой миграции `check-integrity` будет падать на каждом rule без example.

## Контекст
Схема (R13) объявляет у `rule.out_refs` обязательную связь `examples → example, cardinality=many, required=true`. Сейчас в живых docs (`DocsWalker.yml`, `Стек.yml`, `Правила оформления.yml`) есть rule'ы, у которых:
- Под секцией с этим rule **есть** примеры — тогда нужно добавить ref `examples: [<id>, ...]` в YAML rule.
- Под секцией **нет** примеров для этого rule — нужно создать минимальный example рядом и связать.

## Файлы
- `docs/DocsWalker.yml` — пройти все секции, для каждого `rule`/`may_rule` (если may_rule тоже получит обязательный examples — см. ниже) проставить связь.
- `docs/Стек.yml` — то же.
- `docs/Правила оформления.yml` — то же.
- `docs/.docswalker/sequence.txt` — пересчитать максимум, если новые example-узлы добавлены.

## Открытый вопрос перед стартом
**`may_rule` тоже требует example?** В R13 связь добавляется только у `rule`. Если решено распространить на `may_rule` — добавить ref в `may_rule.out_refs` в R13 параллельно и здесь мигрировать. **Default: только `rule`**, потому что `may_rule` — допускающее правило, пример не всегда осмыслен; решение пользователя.

## Действия
1. Утвердить, распространяется ли требование на `may_rule`.
2. Прочитать каждый docs/*.yml; собрать таблицу: `rule_id, section, has_existing_examples_nearby`.
3. По таблице — для каждой rule:
   - Если рядом есть подходящий example в той же секции — связать (добавить `examples: [example_id]` в out_refs rule в YAML).
   - Если нет — сочинить минимальный example (1 строка, описывает применение правила); добавить узел в секцию, обновить sequence.txt, связать.
4. Прогнать `check-integrity` — должен пройти зелёным.
5. Закоммитить.

## Тесты
Этот шаг — миграция данных, не код. Покрывается интеграционным `check-integrity` после миграции. Если хотя бы один rule остался без example — миграция не завершена.

## Риски
- Сочинённые example'ы должны быть осмысленные — не «example: TODO». Если для какого-то rule пример действительно нечего написать — это сигнал, что само правило формулировать стоит иначе. Пометить в чате при ревью.
- При добавлении новых узлов sequence.txt пересчитывается; убедиться, что новые id не конфликтуют с существующими.
