# stg-0003 — update-stale-cli-examples

## Цель
Привести упоминания параметра `move-node` в `docs/DocsWalker.yml` к актуальному API (`--to`, snake_case `to`). Сейчас в трёх местах фигурирует устаревшее `--new-path`/`new_path`, что вводит LLM в заблуждение: реальная CLI-команда — `move-node --id=X --to=Y`.

## Файлы
`docs/DocsWalker.yml` — три точки правки в трёх разных секциях.

## Действия

1. Узел `(#72) move_node` в секции «Операции записи» (definition).
   Поправить text: вместо «`new_path` (id нового родителя; должен входить в `path_targets` типа переносимого узла)» — «`to` (id нового родителя в указанном дереве; для дерева `path` должен входить в `path_targets` типа переносимого узла)». Дополнить упоминанием параметра `tree` (опц., имя tree-scope; по умолчанию `path`), чтобы определение API соответствовало реальной сигнатуре команды (`Commands.cs` для `move_node`: `id`, `to`, опц. `tree`).
2. Узел `(#159) --key=value` в секции «CLI-интерфейс» (rule).
   Поправить text: в перечислении примеров kebab-case-ключей заменить `--new-path` на `--to-id` (актуальный пример из живого CLI — у `create-ref`/`delete-ref`). Остальную формулировку правила не трогать.
3. Узел `(#73) Перенос узла к новому родителю` в секции «CLI-интерфейс» (bare-command example).
   Поправить text: `docswalker move-node --id=42 --new-path=8` → `docswalker move-node --id=42 --to=8`.

## Риски
Узел `(#214) move-node` в секции «Операции записи» уже использует `--to=8` — пересечения с правкой нет, дубль не возникнет. Других упоминаний `--new-path` в `docs/` нет (проверено grep'ом).
