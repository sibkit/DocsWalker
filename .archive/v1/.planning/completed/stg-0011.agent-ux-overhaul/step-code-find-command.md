# stg-0011 — code-find-command

## Цель

Реализовать в kernel новую read-команду `find` — структурный поиск узлов по пересечению multi-tree-фильтров и типов, без текстового запроса. Параметры через MCP arguments: `in_tree` (массив `{name, under}` — для каждого классификатор-дерева указывается id корневой категории, под которой искать), `type` (фильтр по типу узла), `limit` (default 50), `compact` (alias для урезанных полей).

## Файлы

- Kernel: `FindHandler.cs`.
- Command registry.
- `DocsWalker.Mcp/Tools/...` — tool registration + inputSchema.

## Действия

1. Реализовать handler: пересечение наборов узлов, попадающих под каждый `in_tree:{name, under}` (через предвычисленный или on-the-fly индекс tree-scope потомков).
2. Применить фильтр по `type`.
3. Применить `limit`.
4. Применить `compact` — урезать поля до `id, type, title`.
5. Зарегистрировать команду в kernel-dispatcher.
6. Зарегистрировать tool в MCP с inputSchema (массив объектов для `in_tree`).
7. Unit-тесты: пересечение, пустой результат, фильтр по типу, лимит.

## Риски

Производительность пересечений на больших корпусах. На 337 узлах — мгновенно, но логика должна быть индексной (не O(N*M) по узлам и фильтрам).
